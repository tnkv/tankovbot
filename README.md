# 🎲 TankovBot (Телеграм-бот "Кок")

Развлекательный Telegram-бот с механикой игры, в которой пользователи ежедневно крутят рулетку, изменяющую размер их воображаемого агрегата. 

Бот обладает огромной базой случайных ивентов (более 100 штук), которые срабатывают при определенных условиях (размер, дата, случайность).

## 🌟 Фичи
- **Случайные ивенты (Data-driven):** Более 100 разнообразных ивентов (праздничные, зависящие от размера, постироничные, ультра-редкие и механики камбэка для неудачников).
- **Анимация ролла:** Эффект прокрутки числа перед показом результата, работающий в фоне (не блокирующий обработку других сообщений).
- **Продвинутое кэширование:** Интеллектуальный менеджер состояний для снижения нагрузки на SQLite через фоновые батчи.
- **Безопасные миграции:** Использование Alembic для версионирования структуры базы данных.
- **Интеграционные тесты:** Полное покрытие сложной системы ивентов с помощью `pytest`.

---

## 🛠 Технологии
- **Python 3.11+**
- **Aiogram 3.x** (Асинхронный фреймворк для Telegram API)
- **SQLAlchemy + aiosqlite** (ORM и асинхронный драйвер базы данных)
- **Alembic** (Управление миграциями БД)
- **Pydantic Settings** (Управление конфигурацией через `.env`)

## 🚀 Деплой и Установка

### 1. Клонирование и зависимости
Склонируйте репозиторий и установите зависимости:
```bash
git clone https://github.com/tnkv/tankovbot.git tankovbot
cd tankovbot

python -m venv venv
# Активация venv (Windows):
venv\Scripts\activate
# Активация venv (Linux/macOS):
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Настройка окружения
Скопируйте пример файла конфигурации и заполните его своими данными:
```bash
cp .env.example .env
```

Обязательно заполните `BOT_TOKEN` и списки администраторов/заблокированных (в формате JSON массива, например: `[12345, 67890]`).

## 🗄 Накатывание Миграций БД (Alembic)

Проект использует Alembic для управления структурой SQLite базы. 
В зависимости от того, разворачиваете ли вы бота с нуля, или обновляете существующую продакшен-базу, процесс будет немного отличаться.

### Вариант А: Запуск с чистого листа (Новая база)
Если у вас еще нет файла `users.db` и вы запускаете бота впервые:
```bash
alembic upgrade head
```
*Эта команда создаст пустую базу данных и накатит все таблицы сразу.*

### Вариант Б: Обновление существующей PROD-базы
Если у вас **уже есть** боевая `users.db`, которая была создана ДО внедрения системы кэширования (в ней есть только таблица `tgusers`), вам **обязательно** нужно выполнить следующие шаги, иначе миграция завершится ошибкой!

1. Сначала скажите Алембику, что изначальная структура (создание юзеров) уже присутствует в базе:
```bash
alembic stamp 47fff4f82db3
```
*(47fff4f82db3 — это хэш первой/базовой миграции)*

2. Теперь накатите новые изменения (таблицу кэширования):
```bash
alembic upgrade head
```
*(База успешно обновится, сохранив старые данные).*

## 🧪 Запуск тестов

Чтобы убедиться, что система ивентов и логика роста работают корректно, запустите тесты:
```bash
pytest
```
Тесты проверяют инфраструктуру ивентов, а также прогоняют все 100+ ивентов в системе через Smoke-тест (на отсутствие синтаксических ошибок в условиях).
